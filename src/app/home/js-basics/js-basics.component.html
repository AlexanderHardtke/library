Tipp 6

"null" und "undefined"

"undefined" bedeutet das der Wert nicht zugewiesen worden ist oder nicht existiert
"null" wurde als Wert definiert ist aber absichtlich leer

Tipp 7

"==" vs "==="

"==" definiert loose Gleichheit also z.B null == undefined oder 6 == '6' oder true == 1
"===" definiert strikte Gleichheit und checkt ob Wert + Typ gleich sind

Tipp 8

Promise

Promise erwartet mit einer asynchronen Funktion darauf das ein bestimmter Parameter zurückgegeben wird bevor der Code fortgeführt wird. .then oder try and catch.

Tipp 10

let vs const vs var

let definiert nur über die Funktion bis die geschweifte Klammer geschlossen wird was der Wert ist bei var ist der Wert überall definiert. Const ist eine Konstante und kann nicht geändert werden, nur ausgelesen.
